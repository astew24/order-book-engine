A small matching engine that shows how a limit order book works.

This project implements price-time priority matching in Python. The demo below is a simplified browser view of the same ideas: resting liquidity, marketable orders, spreads, and fills.

30 second version

What it models

Limit orders, market orders, cancels, price-time priority, and fills.

How it stores the book

Sorted bid and ask price levels, with FIFO queues inside each level.

What it is for

Learning market microstructure and showing clean trading-systems code in Python.

Send orders

Use these buttons to see how incoming orders change the book.

  • A buy limit order rests on the bid side unless it crosses the best ask.
  • A market buy consumes the cheapest asks first.
  • A market sell consumes the highest bids first.
  • Orders at the same price are matched in arrival order in the Python engine.

Honest limits

This is a single-threaded Python learning implementation. It does not include exchange gateways, risk checks, auctions, persistence, hidden liquidity, or production latency work.
